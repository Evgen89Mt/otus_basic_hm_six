# otus_basic_hm_six

Задание реализация контейнеров.

1. Подзадача разработать аналог контейнера vector. 
    Принципт хранения данных - динамический (работает с кучей, нужно грамотно освобождать ресурсы). Он последовательный, т.е. память устроена, как в массиве "единый блок", можно использовать "адресную арифметику".

    Преимущества:   - быстрый доступ к памяти оператором [].
    Недостатки:     - медленный поиск(O(n)).  


2. Подзадача разработать аналог контейнера list.
    Принципт хранения данных - динамический. (работает с кучей, нужно грамотно освобождать ресурсы) Устройство блок памяти Node, у контейнера есть head и tail. "Адресная арифметика" не работает, связь меду node указатели. Освобождать память пройтись по всему списку. 
    
    Преимущества:   - быстрая вставка и удаление (переопределить указатели).
    Недостатки:     - медленный поиск(O(n)).
